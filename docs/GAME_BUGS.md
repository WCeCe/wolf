## GB-001：放逐无遗言，导致神职信息蒸发

| 字段 | 内容 |
|------|------|
| **优先级** | P0 |
| **状态** | **已修**（v0.2） |
| **现象** | 2 号预言家 R1 被放逐；日志仅 `2 号玩家被放逐`，**无遗言阶段**；之后全场反复讨论「2 号未点名的查杀」，**无人知晓验的是 1 号狼** |
| **规则预期** | 常见 12 人局：**被投票出局者有遗言**（至少一句）；预言家可在遗言中报验人/身份 |
| **原代码问题** | 放逐后无发言，神职信息随玩家死亡从 public 记忆消失 |
| **已实现** | `_elimination_last_words`：系统公告 → `RoleHandler.last_words`（phase=`last_words`）→ `publish_global` speech → `record_last_words` 写入 ledger；**在猎人开枪之前** |
| **关联文件** | `game/phases/voting.py`、`roles/base.py`、`config/prompts/actions/last_words.yaml`、`game/ledger.py`、`memory/selection.py` |

## GB-002：预言家白天发言未强制报验人

| 字段 | 内容 |
|------|------|
| **优先级** | P3 |
| **状态** | **按设计**（不单独修） |
| **现象** | 白天可跳预但不说验谁（藏验策略） |
| **产品结论** | **白天不强制报验人**；信息可在 **放逐遗言** 中交代（见 GB-001 已修）。若遗言仍不说，属模型/玩家选择，非规则漏洞 |
| **关联** | `last_words.yaml` 对预言家：「可说明查验，也可不说」 |

## GB-007：白天发言超长（5.4 未生效）

| 字段 | 内容 |
|------|------|
| **优先级** | P1 |
| **状态** | 已确认 |
| **现象** | 5.4 要求约 120～180 字；本局发言普遍 **300～600+ 字**（如玩家 1 首段） |
| **原因** | 仅 prompt + `discuss_max_tokens: 180`；模型常不遵守；**无写时截断**（刻意不做） |
| **影响** | user token 爆炸；GB-003 开枪公告被淹没；对局拖沓、复读 |
| **修复方向** | ① 加强 task 与 1 条范例 ② 后处理：超 250 字打 warning（已有）→ 考虑 **按句保险丝** ③ 降低 temperature / 换更听话模型 |
| **关联** | [MEMORY_5.4_PLAN.md](./MEMORY_5.4_PLAN.md) |

## GB-008：公聊内容高度同质化

| 字段 | 内容 |
|------|------|
| **优先级** | P2 |
| **状态** | 已确认 |
| **现象** | 多人反复「一边倒踩 2 号」「内耗」「观望」「别带节奏」；R2+ 复读「3 号刀口 + 2 号查杀未消化」 |

| **关联** | `config/prompts/system/*.yaml`、`discuss.yaml` |
